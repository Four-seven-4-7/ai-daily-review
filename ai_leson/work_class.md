## 1课作业

### 1.1：【初阶作业】痛点挖掘机（产品思维练习）

* **目标：** 像产品经理一样，从琐碎的生活/工作中通过“体感”发现真实的刚需。

* **作业内容：** 请列出你在过去一周内，感到\*\*“麻烦、低效、反复、不得不做”\*\*的 3 个具体场景。

* **思考清单：** 针对每一个场景，请回答：

  1. **用户是谁：** 是你自己，还是某个特定的群体（如：财务、直播间运营、独立开发者）？

  2. **当前痛点：** 现在的做法是什么？哪一步让你最想“摔键盘”？

  3. **用户真实需求：** 剥离掉你脑子里的“工具形状”。（*示例：我不是想要一个自动录制软件，我只是想要一份剪辑好的原片。*）

* **提交要求：** 选出你认为最值得解决的一个场景，用一段话描述它。

### 1.2：【进阶作业】MVP 架构师（工程思维切换）

* **目标：** 像工程师一样，思考如何利用现有的“乐高组件”快速、低成本地交付价值。

* **作业内容：** 针对你初阶作业中选出的“最值得解决的场景”，设计一个 **MVP（最小可用产品）** 方案。

* **设计要点：**

  1. **核心链路：** 这个产品最核心的功能是什么？（只写一个，多一个都算违规）。

  2. **组件选择：** 查阅 Part 5 的乐高清单，你会用到哪些工具？（例如：用 Enter.pro 做壳，用 Supabase 存数据，用 Vercel 分发）。

  3. **避坑指南：** 预判一下，在这个方案中，什么因素可能导致用户“不买账”或“没法用”？（回想老师讲的电脑休眠案例，思考你的产品会受限于环境、设备还是人的习惯？）。

* **提交要求：** 形成一份简单的“产品构思图”或文字描述。



## 2课 课后作业

### 2.1初阶作业：想清楚再动手（产品设计 PRD）

**核心目的**：做之前先想好。

* **作业题目**：**《一份价值百万的 PRD 草稿》**

* **完成要求**（三选一，丰俭由人）：

  1. **挑战题（推荐）**：挖掘你生活/工作中的一个真实痛点（比如：记账太麻烦、周报生成器、今天吃什么），模仿课程中的格式，写一份需求文档。

  2. **开放题**：设计一款 **「动态个人主页（Profile）」** 。注意：不仅仅是展示静态信息，需要包含一个互动模块（如：留言板、扫码加好友、简历下载统计）。

  3. **复习题**：如果实在没思路，可以把课上的 **「Credits 申请系统」** 的需求逻辑用自己的话重新梳理一遍，看看有没有逻辑漏洞可以补充。

* **提交格式**：

  * 一份文档链接（飞书文档），格式参考课上提到的 PRD 示例。

### 2.2进阶作业：让数据流动起来（全栈实现）

**核心目的**：验证本节课核心技能——**Supabase 数据库接入**。只写前端那是“画皮”，接了数据才是“画骨”。

* **作业题目**：**《Hello, Real World》**

* **完成要求**：

  1. **实现 PRD**：将你在初阶作业中设计的需求，通过 Enter.pro 或其他工具变成真实可用的网站。

  2. **拒绝假数据**：严禁把数据写死在代码里！必须连接 **Supabase**。

     * 如果是“Credits 系统”或“痛点工具”：必须能提交数据并入库。

     * 如果是“个人主页”：互动模块（如留言、点击计数）的数据必须存通过 Supabase 存取。

  3. **成功发布**：生成公网 URL，确保在你的手机上也能正常访问和操作。

* **提交格式**：

  * 产品的 **公网 URL**。

  * 一张 **Supabase Table Editor** 的截图（证明你确实存进去了数据）。

### 2.3高阶作业：产品经理的自我修养（运营与体验）

**核心目的**：这才是 Vibe Coding 的完全体。一个好的产品，必须要有**数据反馈**，必须**适配手机**，必须要**好用好看**。

* **作业题目**：**《打造属于你的“正式版”产品》**

* **完成要求**（必须完成前两项，第三项为加分项）：

  1. **数据之眼（必须）**：参考课程内容，成功接入 **Google Analytics** 或 **PostHog**，并配置至少一个**自定义事件**（如：`click_submit` 或 `download_resume`）。

  2. **掌上体验（必须）**：完成**移动端适配**。请拿出你的手机访问生成的链接，确保表单、按钮、文字排版在手机上体验丝滑，没有错位。

  3. **Vibe 升级（加分项）**：

     * **设计感**：不要使用默认的黑白灰，尝试通过 Prompt 让 AI 帮你设计一套有格调的 UI（如：毛玻璃特效、渐变风格、大卡片设计）。

     * **功能探索**：在原有 PRD 基础上，自主探索并增加一个新功能（例如：给 Credits 系统加一个“邮件通知”的伪代码逻辑，或者给个人主页加一个“博客列表”）。

* **提交格式**：

  * 产品的 **公网 URL**。

  * **数据看板截图**：展示 GA 或 PostHog 中捕获到的真实用户访问或事件数据。

  * **手机端访问截图**：展示你的产品在手机上的样子。

## 3课课后作业

### 3.1初阶作业：让产品“活”过来（接入 LLM/Agent）

**核心目的**：体验 API 的魅力，学会如何在 Vibe Coding 模式下，通过简单的 Prompt 让 Agent 帮你写好“调用第三方模型”的代码，让产品具备智能交互能力。

* **作业题目**：**《我的第一个 AI Native 功能》**

* **完成要求**：

  * **场景选择（三选一）**：

    * 场景 A（延续）：延续 EP1 或 EP2 的作业，在你的 MVP 产品里尝试接入 AI 能力。

    * 场景 B（创新）：设想一个专门为了 AI 的应用并落地。

    * 场景 C（复刻）：找一个你心仪很久的 AI 小工具，复刻它。

  * **模型接入**：通过 Vibe Coding 接入任意一个主流大模型 API（DeepSeek, 豆包、GLM、Gemini, Claude...）。**关键点**：学会处理 API Key（环境变量配置）和 Edge Function（函数调用）。

  * **Prompt 调优**：编写 System Prompt（系统提示词），规定 AI 的角色（如：“你是一个严格的财务审批员”或“你是一个幽默风趣的 Chris 替身”）。

* **提交格式**：

  * **公网 URL**。

  * **交互截图**：展示 AI 正在回答问题或给出审批建议的瞬间。

  * **System Prompt**：复制你写在代码里的那段提示词。

### 3.2进阶作业：接入更多可能（接入Workflow/第三方服务）

**核心目的**：跳出单纯的“对话”，让产品具备“业务执行力”。学会让你的网站去指挥其他软件干活（Automation）。

* **作业题目**：**《自动化：让系统替我打工》**

* **完成要求**：

  * **需求场景（三选一）**：解决一个需要跨平台协作的痛点。

    * *场景 A（工具流）*：延续 EP1 或 EP2 的作业，在其中接入你需要的第三方平台服务，让你的产品能够给用户提供更多服务。例如：在你的个人主页显示你所在城市的实时天气和心情指数。

    * *场景 B（通知流）*：当有新 Credits 申请或新留言时，不仅存入数据库，同时触发 **飞书/钉钉/Telegram** 机器人，给你的手机发一条通知。

    * *场景 C（工作流）*：接入 **Coze** / **Dify&#x20;**&#x7B49; Agent 编排平台的 API。例如：用户输入一个主题，你的网站调用 Coze 的工作流生成一张海报，并返回图片展示在页面上。

* **提交格式**：

  * **公网 URL**。

  * **效果证明**：

    * 如果是通知流：截图手机上收到的飞书/Telegram 消息。

    * 如果是工作流：截图网页上生成的内容。

  * **架构草图**（可选）：简单画一下数据是怎么流转的（如：Web -> Next.js API -> Coze -> Web）

1. 如果代码跑不通，找到错误信息并把把完整错误复制给它，说：‘报错了，请根据这个错误修正代码’。

2. 记得 API Key 永远不要暴露在前端代码里，要放在**环境变量**（Environment Variables）中。

3. 一个产品的构成是很多外部服务的集合，跟 EP1 中提到的，**产品其实是可以拼出来的**。
