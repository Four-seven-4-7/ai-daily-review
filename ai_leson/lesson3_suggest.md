## 3. 来看看作业

### 是什么

打开主文档查看作业 [ 第3课：与时俱进做出你的第一款 AI 应用](https://waytoagi.feishu.cn/wiki/IowDwmesfiXqGzkmVsrc4Qzzndf#E0DOdNjRtoPKNsxojLfc4vmCnEc)，会发现这次作业还是分两个层级。

**初阶作业**的核心交付不是功能堆砌，而是一个能被公开访问的 URL、一次真实的智能交互瞬间截图、以及你写在代码里的 System Prompt（用来固定角色与边界）。

选题上不需要纠结“创新 or 复刻”，更建议沿用你用前面两节课的东西做加法：在原本能跑的 MVP 上加一条“调用模型/智能体”的能力，完成一次从页面点击到后端函数到模型返回再渲染到页面的闭环。

**进阶作业**的本质是把“会聊天”升级成“会办事”，也就是让网站去触发第三方服务或工作流平台做执行，这会天然带来更多鉴权、回调、异步与跨平台链路问题，提前预期到这一点反而更稳。

### 做哪个、怎么做

> 展望一下，“我现在的能力、时间、风险承受力，到底该把作业做到什么复杂度。”

如果前两节课没补完，尤其第二课还没把“用 Enter 做出网页—连接 Supabase—发布上线”这条链路跑顺，那更建议先回去补课再选题，因为第三课会默认你已经具备这些基础动作，否则你会在“到底哪里出问题了”这件事上反复消耗时间。



【**初阶作业】**&#x60F3;稳稳交作业、并且第一次体验“产品真的连上 AI 服务”的人，优先选第一道，也就是“我的第一个 AI Native 功能”。

* 它的训练重点很明确：你需要把外部模型或智能体的能力，通过后端函数接进你的网站里，并且学会把 Key 放在安全的位置（课程示例是把密钥放进 Supabase 的 Secrets/Keys，给 Edge Function 调用），最后在页面上看到一次真实的智能交互，然后把公网 URL、交互截图、System Prompt 这三样东西交上去就够了。

* 这道题最容易踩的坑也相对集中，基本都围绕“鉴权没配对、环境变量没读到、接口参数不合法、以及需要学会从 Network 里把红色报错完整复制出来让 Agent 按错误修”。



【**进阶作业】**&#x7B2C;二道“自动化：让系统替我打工”更适合已经把第一道跑通、并且愿意面对跨平台链路的人，因为它会把难度从“调用一次模型”升级成“你的网站要去指挥别的软件干活”。

* 课堂里其实已经给了很典型的延展方向：你可以在原项目里接更多第三方能力（例如天气这类外部数据），也可以做通知流，让网页收到新申请后触发飞书等消息通知，或者接入 Coze/Dify 这类编排平台，让网页调用工作流返回海报等结果。

* 这道题关键通常不是页面写得多漂亮，而是链路是否真的跑通、证据是否能自证（比如通知截图、网页生成结果截图），以及你能不能把每一步的鉴权与数据流向讲清楚。

## 4. 其他注意事项与建议

### 你的Key只属于你自己，不要共享给任何人

Key能消耗你的Token，也可以用于未经授权随意访问阿里云的其他服务。Key 永远不要直接写在前端代码，也不要因好心而发给其他同学使用！ 这是一个原则问题。

如果你让Vibe Coding软件直接在前端用 Key，它可能会好心提醒你，也可能会因为安全策略直接报错。把 Key 藏在后端的 Secrets 里，既安全又专业。

### 关于购买域名与网站上线

如果你打算买域名，别只盯着阿里云这一类国内注册商，因为现在国内域名购买也要实名认证，审核需要几个工作日，可能拖慢作业节奏；想省时间可以考虑国外注册商，比如 [https://www.namesilo.com/](https://www.namesilo.com/%E3%80%81https://www.godaddy.com/zh) [https://www.godaddy.com/zh](https://www.namesilo.com/%E3%80%81https://www.godaddy.com/zh) 这类。
不想花钱也可以找提供免费二级域名的服务，只要**支持 CNAME 接入**就能用；这类服务选择很多，自己搜一圈挑一个自己觉得靠谱点的即可。
域名这件事别当作“锦上添花”，后面的课程大概率还会继续用到，提前准备好会让你后续省心。

### 网站出了问题怎么办

如果没有直接显示错误信息，可以去浏览器开发者工具的 Network 或者下面控制台里找红色请求、复制完整报错给 Agent 修。开发过程中针对顽疾问题，可以告诉AI将流转步骤打印在控制台里，然后拿着控制台打印出来的东西复制给AI看。
