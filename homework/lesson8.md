## 8.1 项目路演
*   **背景**：针对 flomo 碎片记录难以复盘的真实痛点。
*   **成就**：利用 AI 编程，在 0 前端基础的情况下，独立完成了从 UI 设计、Supabase 全栈连接、到 GitHub Actions 自动化链路的开发。
*   **架构演进**：
    1. **V1.0**：实现基础的 MVP 功能。
    2. **V2.0**：上线 AI 智能分析洞察。
    3. **V3.0**：上线飞书提醒工作流。

## 8.2 学习复盘
*   **踩坑记录**：在开发“数据同步”逻辑时，曾因过度追求功能导致本地与云端冲突。这次经历让我明白“需求蔓延”的危害。
*   **心态转变**：从“想做全能工具”到“死磕一个核心痛点”，心态的转变是最大的收获。

## 8.3 上线部署
*   **托管平台**：GitHub。
*   **部署平台**：Vercel。
*   **自动化流水线**：实现了推送代码即自动部署。

## 8.4 学习总结与未来展望
1. **精准 Prompt 是核心**：作为 90% 以上代码由 AI 协助完成的开发者，我意识到清晰的需求描述比写代码更难、更重要。
2. **安全防护意识**：经历了 GitHub 密钥拦截事件后，我深刻理解了 API 安全管理的重要性。
3. **长期主义**：结课不是终点，我将持续迭代这个复盘工具，真正解决自己生活中的低效环节。
