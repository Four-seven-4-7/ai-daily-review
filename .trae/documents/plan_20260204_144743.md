# Vercel 双页部署与链接保护方案

您担心“原链接被串夺”，实际上是担心新页面覆盖了原有的主页。
我的方案将采用 **“双页并存”** 策略：保持 `index.html`（App）作为默认主页不变，将 `intro.html`（介绍页）作为独立子页面存在。

## 1. 核心策略：双入口互通
*   **主页 (App)**: 保持文件名 `index.html` 不变。
    *   访问地址：`https://您的项目.vercel.app/`
    *   **安全承诺**：您的原链接不会有任何变化，用户访问域名依然直接进入 App。
*   **介绍页 (路演)**: 保持文件名 `intro.html` 不变。
    *   访问地址：`https://您的项目.vercel.app/intro`
    *   **路演用法**：您在路演时，直接分享这个带 `/intro` 的链接即可。

## 2. 执行步骤

### 第一步：创建 Vercel 配置文件 (vercel.json)
我们将创建一个配置文件，开启“干净链接”功能，这样访问 `/intro` 时自动加载 `intro.html`，既美观又无需输入 `.html` 后缀。

### 第二步：修改 intro.html 内部链接
您现在的 `intro.html` 里写死了 `github.io` 的链接。为了让它在 Vercel 上能顺滑跳转到您的 App，我们需要将这些链接改为 **相对路径**。
*   **修改前**: `<a href="https://four-seven-4-7.github.io/ai-daily-review/">`
*   **修改后**: `<a href="/">` (直接跳转到当前网站的主页)

### 第三步：部署
由于您之前已经配置了 Vercel（或 GitHub Actions），您只需要确认修改并提交代码，Vercel 会自动识别这两个文件并同时部署。

## 3. 预期结果
| 页面 | 访问 URL | 说明 |
| :--- | :--- | :--- |
| **主页 (App)** | `...vercel.app/` | 原有链接，完全不受影响 |
| **介绍页** | `...vercel.app/intro` | 新增的路演页面 |
