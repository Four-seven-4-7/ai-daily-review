# 极简 MVP 方案 (No-Auth, Manual Sync)

收到，这是一个非常明智的决策。我们回归核心价值，剔除所有导致不稳定的复杂度（自动同步、登录认证、网络检测等），打造一个**纯粹、快速、稳定**的复盘工具。

## 核心变更点
1.  **移除登录系统**：
    *   删除所有登录/注册界面代码。
    *   恢复“人人可用”模式（但为了数据安全，我会保留之前数据库的 RLS 策略，并生成一个公共的匿名 User ID，或者干脆回退到无 RLS 模式，为了最快 MVP，建议**回退到无 RLS 模式**，即所有拿着 Key 的人都能存取数据，但既然是个人 MVP，这完全可以接受）。
    *   *修正：为了兼容您已经运行的 SQL（开启了 RLS），我会在代码里硬编码一个固定的“公共用户 ID”，或者指导您去 Supabase 关闭 RLS。为了极简，建议代码里生成一个随机 ID 存本地 LocalStorage 作为“用户身份”，实现免登录但有隔离。*

2.  **移除自动同步/自动保存到云端**：
    *   **本地优先**：所有数据默认只存在 LocalStorage（浏览器缓存），速度最快，永不卡顿。
    *   **手动云端交互**：
        *   **保存**：只有点击“提交复盘”按钮时，才尝试推送到 Supabase。如果断网，提示“仅保存到本地”。
        *   **读取**：打开页面默认只看本地。增加一个“从云端拉取”按钮，只有点击时才去联网查数据。

3.  **断网数据处理**：
    *   正如您所说，断网时产生的数据不需要强求保留到云端，只要本地有就行。

## 执行计划
1.  **代码清理**：删除 `AuthScreen` 组件，移除 `checkSession` 等副作用代码。
2.  **身份逻辑简化**：使用 `localStorage` 中存储的一个随机 UUID 作为 `user_id`，实现“隐式登录”，既不需要输入密码，又能满足数据库的 `user_id` 字段要求。
3.  **交互逻辑调整**：
    *   初始化：仅读取本地 LocalStorage。
    *   提交按钮：保存本地 + 尝试推送云端（失败仅提示不阻断）。
    *   增加“同步”按钮：手动触发云端拉取。
