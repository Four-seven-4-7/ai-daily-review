# 优化 LCP (最大内容绘制) 方案

您反馈的 LCP 高达 21.10 秒，主要是因为页面内容（特别是 React 渲染的 DOM 元素）需要等待巨大的 JS 库（React, Ant Design, Babel）下载并执行完毕后才能显示。

## 问题根源
1.  **客户端渲染 (CSR) 延迟**：浏览器必须先下载 5MB+ 的 JS 库，然后编译 JSX，最后才生成 HTML。在此之前，用户看到的是白屏或加载圈。
2.  **资源阻塞**：Ant Design 的图标库和样式文件较大，阻塞了渲染。

## 解决方案
我们将采用 **"骨架屏 + 资源预加载"** 策略，让用户在 JS 加载完成前先看到一个静态的页面框架，从而将 LCP 降低到 1-2 秒。

### 具体措施
1.  **添加静态骨架屏 (HTML Skeleton)**：
    在 `<div id="root">` 内部直接写一段静态的 HTML/CSS 代码（模拟标题、输入框的灰色占位符）。这样浏览器下载完极小的 HTML 文件后，立刻就能显示内容，无需等待 JS。
    *   *效果：用户点开即看到页面轮廓，心理等待感降低。*

2.  **优化资源加载顺序**：
    *   将核心 CSS (`antd.min.css`) 放在 `<head>` 最前面。
    *   使用 `defer` 属性异步加载非关键 JS，避免阻塞 HTML 解析。

3.  **移除 Babel 实时编译 (进阶)**：
    *   目前我们用的是 `babel-standalone` 在浏览器里编译 JSX，这是性能杀手。
    *   *权衡*：为了保持“单文件极简模式”，我们暂时保留 Babel，但通过骨架屏来掩盖编译延迟。
